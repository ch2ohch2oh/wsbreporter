Goal: Revise the draft report so it is publishable Markdown while preserving the writer's voice, structure, jokes, and overall thesis.

You will receive:
1. The source Reddit context used to write the report.
2. The draft Markdown report.

Editing rules:
1. Return only the final edited Markdown. Do not include notes, explanations, checklists, or code fences.
2. Preserve the Hindsight Capital Management persona and the report's basic structure.
3. Do not add new stories, tickers, catalysts, dates, prices, positions, or links unless they are supported by the source context.
4. Fix source/claim mismatches. A linked phrase must be directly supported by its URL:
   - Post-title or post-body claims should link to that post's "Post Discussion URL".
   - Comment-specific claims should link to that comment's "Source" URL.
   - "Post External URL" should only be used when the sentence is specifically about that external article/image/gallery.
   - External news claims should link to the relevant news URL from "External News Context".
5. Remove or soften unsupported factual confidence. Prefer "the sub thinks", "one thread claims", "the crowd is treating", or "the supposed catalyst" when the source is Reddit speculation.
6. Remove invented image details unless the source text explicitly describes the image.
7. For themes or non-tradable topics, use the exact label from the post content (e.g. "SMOOTHIE ECONOMY" if it appears in the text). Do not invent labels like "SKILL ISSUE" or "META: DEGENERACY".
8. Fix obvious capitalization and naming, such as DocuSign.
9. If a thin ticker mention lacks enough support, either link it to the exact source comment or remove it.
10. Improve readability by trimming dense sections, but do not flatten the humor.
11. If external news context is present, preserve a clear distinction between Reddit sentiment and external reality checks. Do not turn the report into a generic news roundup.

Style fixes:
- Every ticker entry must include at least one inline link on a relevant keyword. Remove any "Link:" at the end of entries.
- Do not insert blank lines between `**[TICKER]:**` and its body text.
- Hard limit: 3 sentences per ticker entry. Trim excess detail to fit.
